# 知识库集成说明

AgentOS 默认把当前 `My-Knowledge-Base` 作为 Obsidian 知识库根目录。

Agent 脚本应把输出 Markdown 写入约定目录，例如：

- `04_Resources/AI-News/Daily`
- `04_Resources/AgentOS/Knowledge-Organizer`
- `04_Resources/AgentOS/Prompt-Curator`
- `04_Resources/AgentOS/Error-Review`
- `04_Resources/AgentOS/GitHub-Backup`

Runner 执行完成后会扫描 Agent 的 `output_path`，识别 Markdown 文件，并写入 `KnowledgeAsset` 索引。

AgentOS 不删除、不重命名、不移动知识库内容。文件整理仍由你确认后执行。
