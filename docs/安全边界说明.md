# 安全边界说明

AgentOS v1 的安全边界：

- 不执行未注册命令
- entrypoint 必须在允许目录内
- 默认只允许 `scripts/` 下的 Python 脚本
- 使用 `subprocess` 时设置 `shell=False`
- 不允许危险 shell token，例如 `&&`、`|`、`;`、重定向等
- API Key 只从环境变量读取
- 不在日志中显示 API Key 明文
- 不自动删除文件
- 不自动重命名知识库文件
- GitHub 备份 Agent 第一版只读取 `git status`，不自动 push
- 不绕过系统或用户权限

如果未来引入 Docker 或远程执行，需要重新设计沙箱、资源限制和权限审批。
